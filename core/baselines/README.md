# Baselines: How To Run

This page documents baseline continual-learning runs (non-CARvE).

Back to CARvE main page: [README.md](../../README.md)
## Setup

From repository root:

```bash
uv venv
source .venv/bin/activate
uv pip install -e .
cp .env.example .env
```

## Main Baseline Config

Primary training config:

- `configurations/train_config.yaml`

Key mode switch:

- `mode` in `configurations/train_config.yaml`

## Baseline Modes


### Only-Retrievers

Use a retriever to augment inputs during training without sampling replayed examples. This is useful when you want the model to condition on retrieved model cards or metadata but not perform explicit rehearsal.

Configuration (example):

```yaml
retriever: bm25                # or sentence_transformer, flagembedding, splade, null
model_indices: ["e1"]         # which model index(s) to use for retrieval
```

Run:

```bash
train-baseline --config configurations/train_config.yaml --variant_name only-retrievers
```

### RAT baseline

Configure `mode: replay` and set `retriever` and `model_indices` to enable RAT. Control how much replay to include with `replay_percentage` or `replay_num_samples`.

Configuration (example):

```yaml
mode: replay
retriever: flagembedding
model_indices: ["e1", "e1_e2"]
replay_percentage: 0.1    # replay 10% of previous data (or set replay_num_samples)
```

Run:

```bash
train-baseline --config configurations/train_config.yaml --variant_name rat-replay
```

Notes:

- `model_indices` should point to files created by `data/generate_model_indices.py` (see data/README.md).

## Evaluate Baselines

```bash
eval-baseline --config configurations/eval_config.yaml
```

Useful eval fields include:

- `lora_adapters`
- `lora_merging_strategy`
- `eval_batch_size`
- `temperature`

### Oracle (Retriever-only) baseline

The Oracle baseline is a retriever-only evaluation mode that selects the target model purely via the retriever (no LoRA model generation). When enabled, the evaluation script returns the `model_name` selected by the retriever as the predicted response.

To run Oracle evaluation, set `lora_adapters` to `null` in `configurations/eval_config.yaml`, configure a retriever and model indices, and enable the `oracle` flag. Example:

```yaml
lora_adapters: null
retrievers: ["flagembedding"]
model_indices: ["e1"]
oracle: true
```

Run:

```bash
eval-baseline --config configurations/eval_config.yaml
```

Behavior notes:

- With `oracle: true` the retriever uses a domain-filtered retrieval (`retrieve_model_card_with_oracle`) which restricts matches to documents whose `domain` matches the test example before ranking.
- With `oracle: false` the retriever performs a normal retrieval across the full index (`retrieve_model_card`).
- `model_indices` should point to files generated by `data/generate_model_indices.py` (see data/README.md).
